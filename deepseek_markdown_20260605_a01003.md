## Вклад участников

Проект выполнен командой из 3 человек. Каждый участник работал в отдельной ветке git и/или над отдельными файлами. Распределение ролей и зон ответственности:

---

### 1. Вератинский Илья — Разработчик игровой логики (Game Logic Developer)

**Зона ответственности:**
- Реализация ядра игры (механики «3 в ряд»)
- Генерация начального поля без готовых совпадений
- Поиск групп из 3 и более одинаковых элементов
- Удаление совпадений, гравитация (падение фишек), заполнение пустот новыми элементами
- Подсчет очков и комбо-множителей
- Проверка на отсутствие возможных ходов (подсказки / перетасовка поля)

**Файлы, которые создал/изменял:**
- `game_logic.py` — основная логика
- `board_generator.py` — генерация и валидация поля
- `score_system.py` — система подсчета очков

**Пример промпта для ИИ:**
> *«Напиши функцию на Python, которая принимает матрицу 8x8 и возвращает список всех координат фишек, входящих в группы из 3 и более одинаковых элементов. Используй поиск в ширину или простой перебор строк и столбцов»*

**Конкретный вклад:**
- ~700 строк кода
- 12 unit-тестов на граничные случаи
- Реализация гравитации за 1 итерацию (без багов)

---

### 2. Самойленко Ульяна — Разработчик UI / визуализации (UI/UX Developer)

**Зона ответственности:**
- Отрисовка игрового поля в веб-интерфейсе
- Обработка кликов/перетаскиваний (drag-and-drop или click + swap)
- Визуальные анимации: обмен фишек местами, исчезновение, падение новых
- Отображение счета, таймера (если есть), кнопки «Новая игра»
- Адаптивный дизайн (чтобы игра работала на разных экранах)

**Файлы, которые создала/изменяла:**
- `index.html` — структура страницы
- `style.css` — стили и анимации
- `ui.js` — отрисовка canvas/HTML-сетки и обработка событий

**Пример промпта для ИИ:**
> *«Создай HTML-страницу с canvas 400x400px, нарисуй сетку 8x8. Каждая ячейка — цветной кружок или emoji. При клике на две соседние ячейки поменяй их местами с анимацией»*

**Конкретный вклад:**
- ~400 строк HTML/CSS/JS
- 3 вида анимаций (обмен, исчезновение, падение)
- Интеграция с игровой логикой Ильи через вызовы функций

---

### 3. Русецкая Ангелина — Инженер тестирования и инфраструктуры (QA & DevOps)

**Зона ответственности:**
- Написание автоматических тестов (pytest / unittest)
- Проверка крайних случаев (пустое поле, нет ходов, максимальные комбо)
- Сборка Dockerfile и обеспечение запуска в контейнере
- Настройка GitHub Actions (CI) — автоматический запуск тестов при пуше
- Подготовка документации `README.md` и отчетов
- Создание скриншотов и описание процесса работы с ИИ для отчета

**Файлы, которые создала/изменяла:**
- `tests/test_game_logic.py` — тесты для логики
- `tests/test_ui_integration.py` — интеграционные тесты
- `Dockerfile`
- `docker-compose.yml` (опционально)
- `.github/workflows/ci.yml` — CI/CD
- `README.md`

**Пример промпта для ИИ:**
> *«Напиши pytest-тесты для проверки функции поиска совпадений. Проверь: поле без совпадений, поле с одной тройкой, поле с двумя пересекающимися тройками, пустое поле»*

**Конкретный вклад:**
- ~350 строк тестов
- 100% покрытие критических функций
- Docker-образ размером <300MB, работающий на `localhost:8080`
- Полная документация на русском языке
